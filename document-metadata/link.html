link

Specifies relationships between the current document and an external resource. Possible uses for this element include defining a relational structure for navigation.

as

Only used when rel="preload" was sent in the element. Specifies the type of content loaded, which is necessary for content prioritization, requesting matches, applying the correct content security policy, and setting the correct Accept request header.

crossorigin

Indicates whether the search for the related image should be done using CORS or not.

anonymous

A cross-origin request is performed. But no credential is sent. If the server does not give credentials to the origin site, the image will be tainted and its usage restricted.

use-credentials

A cross-origin request is performed with credential is sent. Of the server does not give credentials to the origin site, the image will be tainted and its usage restricted.

href

Specifies the URL of the linked resource.

hreflang

Indicates the language of the linked resource. use only if href attribute is present

media

Specifies the media which the linked resource applies to. Its value must be a media query. This attribute is mainly useful when linking to external stylesheets by allowing the user agent to pick the best adapted one for the device it runs on.

rel

Names a relationship of the linked document to the current document. WebTV also supports the use of the value next for rel to preload the next page in a document series.

sizes

Defines the sizes of the icons for visual media contained in the resource. It must be present only if the rel contains the icon link types value.

any

The icon can be scaled to any size as it is in a vector format, like image/svg+xml.

<width in pixels>x<height in pixels> or <width in pixels>X<height in pixels></height>

Each of these sizes must be contained in the resource.

type

Used to define the type of the content linked to. The value of the attribute should be a MIME type such as text/html, text/css, and so on.
